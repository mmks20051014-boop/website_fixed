セクション3: UIエフェクト・アニメーション仕様【neoai.jp参考】

3-1. ヒーローエリア パーティクル・3Dアニメーション

デザインコンセプト：
neoai.jpのようなテック系スタートアップサイトのビジュアル表現を採用

実装仕様 (CSS)：

/* ヒーローエリア レイアウト */
.hero {
  position: relative;
  min-height: 100vh;
  display: flex;
  align-items: center;
  justify-content: center;
  overflow: hidden;
  background: linear-gradient(135deg, #0f1923 0%, #1a2a3a 50%, #0d1b2a 100%);
}

/* キャンバス（Three.js / Particles.js 描画エリア） */
#hero-canvas {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  z-index: 0;
}

/* テキストコンテンツは前面に */
.hero-content {
  position: relative;
  z-index: 1;
  text-align: center;
  padding: 2rem;
}

.hero h1 {
  font-size: clamp(2rem, 5vw, 4rem);
  font-weight: 700;
  color: #ffffff;
  text-shadow: 0 0 40px rgba(100, 180, 255, 0.4);
  letter-spacing: 0.02em;
  margin-bottom: 1.5rem;
}

JavaScript アニメーション実装仕様：

const heroAnimation = {
  type: "particles_3d",        // パーティクル + 3Dメッシュ
  particleCount: 120,          // パーティクル数
  connectionLines: true,       // 粒子間の接続線
  floatingGeometry: true,      // 浮遊する抽象ジオメトリ
  colorPalette: ["#64b4ff", "#8ab4d4", "#ffffff"],
  opacity: 0.6,
  speed: 0.3,
  mouseInteraction: true
};

推奨ライブラリ：

  • three.js ＋ カスタムシェーダー（最高品質・重め）
  • tsParticles （軽量・スマホ対応・推奨★）
  • vanta.js（NET / WAVES エフェクト）

モバイル対応：

@media (prefers-reduced-motion: reduce) {
  #hero-canvas { display: none; }
  .hero { background: linear-gradient(135deg, #0f1923, #1a2a3a); }
}

スマホではパーティクル数を 60 以下に削減

3-2. スクロール連動ヘッダー（Glassmorphism）

neoai.jp参考： スクロール時に背景が半透明＋ぼかし効果（backdrop-filter）

/* デフォルト状態（ページトップ） */
header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  z-index: 1000;
  padding: 1.2rem 2rem;
  background: transparent;
  transition: background 0.4s ease, backdrop-filter 0.4s ease,
              box-shadow 0.4s ease;
}

/* スクロール後（.scrolled クラスをJSで付与） */
header.scrolled {
  background: rgba(15, 25, 35, 0.6);
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px);
  box-shadow: 0 1px 24px rgba(0, 0, 0, 0.3);
  border-bottom: 1px solid rgba(100, 180, 255, 0.1);
}

JavaScript実装：

const header = document.querySelector('header');
window.addEventListener('scroll', () => {
  if (window.scrollY > 80) {
    header.classList.add('scrolled');
  } else {
    header.classList.remove('scrolled');
  }
}, { passive: true });

セクション7: 技術仕様

7-1. 基本構成フロー

HTML5 + CSS3 + JavaScript (Vanilla / 最小限のライブラリ)
↓
レスポンシブデザイン（モバイルファースト）
↓
SEO対策（メタタグ・OGP・構造化データ・サイトマップ）
↓
Core Web Vitals 最適化（LCP / CLS / INP）
↓
アニメーション最適化（GPU描画 / will-change / requestAnimationFrame）

7-2. アニメーション最適化方針

要素 最適化手法
パーティクルCanvas requestAnimationFrame でループ / offscreenCanvas 検討
CSS アニメーション transform / opacity のみ使用（レイアウトシフト回避）
スクロールイベント passive: true オプション付与
backdrop-filter GPU加速を前提にパフォーマンス計測必須

7-3. 必須機能チェックリスト

機能 優先度 備考
✅ モバイル対応（スマホ・タブレット） 必須 モバイルファースト設計
✅ パーティクルアニメーション（Hero） 高 モバイルは軽量版
✅ スクロール連動ヘッダー blur 高 backdrop-filter使用
✅ カードホバーエフェクト 高 枠線発光・グラデーション移動
✅ 検索機能（記事検索） 中 -
✅ お問い合わせフォーム（スパム対策付き） 必須 reCAPTCHA推奨
✅ ソーシャルシェアボタン（X/Facebook） 中 -
✅ ページネーション（記事一覧） 中 -
✅ prefers-reduced-motion 対応 必須 アクセシビリティ配慮
